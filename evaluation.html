Heuristic Evaluation

Nielsen's 10 usability heuristics applied to the SpeakBirth prototype.

Method

How the evaluation was conducted

The SpeakBirth prototype was evaluated against Jakob Nielsen's 10 Usability Heuristics. Each of the three core flows — provider search, benefits navigator, and equity reviews — was walked through screen by screen, with usability issues flagged and rated for severity on a three-point scale:

  • Low — cosmetic problem; doesn't block the user
  • Medium — usability issue worth fixing in the next iteration
  • High — major problem that blocks core task completion or harms trust
Findings

Identified issues

Six usability issues surfaced across the three flows. Each is mapped to a Nielsen heuristic and paired with a suggested fix.

H1 Visibility of system status Medium
When the user submits a review, there is no loading indicator between tapping "Submit" and seeing the confirmation screen. On a slow connection, the user may tap twice and submit duplicate reviews.
Suggested fix: Add a loading spinner on the submit button with the label "Posting…" and disable the button until the response returns.
H2 Match between system and the real world Low
The benefits navigator uses the term "ACA" without explanation. Users unfamiliar with the acronym may not know whether to select it.
Suggested fix: Use the full phrase "Marketplace (Affordable Care Act)" or add a small info icon that explains the term in plain language.
H3 User control and freedom Medium
On the review submission form, there is no "Save as draft" option. A user writing a long, emotional story about a traumatic birth experience could lose their progress if they navigate away.
Suggested fix: Add auto-save behavior on the text field and surface a "Save draft" button. Show a clear confirmation prompt if the user tries to exit without saving.
H5 Error prevention High
The provider search defaults to a 10-mile radius. In rural areas, this may return zero results, leaving the user feeling like there are no providers near them at all — when really, the radius needs adjusting.
Suggested fix: When results are zero, surface a clear empty state with a button to expand the search radius automatically. Default to a wider radius for users whose zip codes fall in rural areas.
H6 Recognition rather than recall Low
The active state on filter chips is visually subtle. A user who applies a filter on the previous screen may forget it's active when they return to the results.
Suggested fix: Add a small badge to the filter icon on the results screen showing the count of active filters (e.g., "3 active").
H10 Help and documentation Medium
First-time users may not understand what "equity rating" means or how it differs from a regular star rating. There is no in-app explanation.
Suggested fix: Add a one-time tooltip on the equity rating that explains what it measures, with a "Learn more" link to a longer explanation of the rubric.
Revisions

Design changes based on evaluation

The two highest-impact issues — the rural empty state (H5) and the missing draft save (H3) — were prioritized for revision in the next iteration of the prototype. The empty state fix is particularly important because rural Black mothers are a target user group, and a zero-results experience could push them out of the app entirely.
